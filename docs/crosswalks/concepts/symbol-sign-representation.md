## Scope Note

This crosswalk asks whether XwkOnt needs a distinct concept for the **sign
vehicle / representation-form / symbolization layer** that shows up across
several source ontologies. It is intentionally narrower than
[`information-artifact.md`](information-artifact.md) and
[`proposition-content.md`](proposition-content.md).

**Narrowed 2026-07-06 (maintainer decision).** Verification against
`information-artifact.md`'s own cited evidence found the following:

- YAMATO's `representing thing` and `representation`/`representation form` quote
  the same passage `information-artifact.md`'s own YAMATO correspondence already
  quotes in full, including the "form and content" line. Excluded from this crosswalk.
- SUMO's `SymbolicString`/`Icon`/`LinguisticExpression` are `rdfs:subClassOf ContentBearingPhysical`,
  the class `information-artifact.md` already crosswalks. Excluded from this crosswalk.
- GFO's `Symbol_structure` and GUM's `Symbolization`/`Signification` are terms `information-artifact.md`
  itself lists as considered and rejected for its own scope (type-side category; relation-type).
  Retained in this crosswalk.
- GUM's `NameOf`/`NameRelation` are relation-types, the same kind as `Symbolization`/`Signification`.
  Retained in this crosswalk.
- GUM's `Name` has a stated definition ("an accepted but possibly arbitrary label for
  some entity") comparable in form to BFO(IAO)'s own Information Content Entity definition
  (generically dependent, stands in a relation of aboutness to some entity). Excluded from
  this crosswalk; not added to `information-artifact.md` by this pass — see Future Work.
- GUM's `NameEvent` ("the process of giving something a name") is a process/event term. Excluded
  from this crosswalk; not checked against `process.md`/`event.md` by this pass — see Future Work.

This crosswalk's current evidence is GFO's `Symbol_structure` and GUM's `Symbolization`/`Signification`/`NameOf`/`NameRelation`.

Core placement remains an open question for review. The concept is clearly
candidate-stage only.
