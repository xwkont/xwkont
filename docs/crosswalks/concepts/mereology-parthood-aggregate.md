## Scope Note

**Selection rationale (per `ADR-0018`'s admission bar):** Mereology / Parthood / Aggregate is the seventh concept drafted from the `0.2.0` batch. Direct primary-source verification this session found a nameable "aggregate/no-unity" category in **4 of 8 sources** (BFO `fiat object part`/`object aggregate`, DOLCE `arbitrary-sum`, SUMO `Collection`, UFO `Collective`) — the same source count `TODO.md`'s seed claimed, but with one substitution: YAMATO's claimed `non-unitary` term was not found anywhere in the fetchable 2010 technical report (see Uncertainty), while UFO's `Collective` — not in the original seed — was found to be a strong, directly quotable correspondence. GFO, TUpper, and GUM were confirmed to have no reified aggregate/collection *entity* class, though TUpper does contribute the parthood *relations* (`spatial_part`, `physical_part`) already documented in `space.md`.

This crosswalk compares each source ontology's own treatment of pluralities and mass-like wholes that lack the unity criterion of an Object/kind-instance — an arbitrary or criterion-bound sum of parts, as distinct from Object (already `reviewed`, which covers unified wholes) and from Quantity/Amount of Matter (already drafted, `quantity-amount-of-matter.md`, which covers homeomerous stuff rather than pluralities of discrete members). Sources compared: BFO, DOLCE, SUMO, UFO, GFO, YAMATO, TUpper, GUM (per `ADR-0015`'s 8-source scope).

**Cross-cutting finding, established this session:** the four sources with a genuine aggregate/collection category split into two sub-patterns by how strict their unity criterion is. BFO's `object aggregate` and UFO's `Collective` both require member parts to play a *uniform* role with respect to the whole (BFO: "a plurality... as member parts which together form a unit"; UFO: "parts play the same role with respect to the whole, e.g., the Black Forest as a collective of trees"). DOLCE's `arbitrary-sum` and, more loosely, SUMO's `Collection` instead allow a genuinely unrestricted, criterion-free sum (DOLCE: "the mereological sum of any two or more endurants... no unity criterion... 'extensional'"; SUMO: "members can be added and subtracted without thereby changing the identity of the Collection"). BFO additionally supplies a second, structurally distinct category with no counterpart in any other source checked — `fiat object part`, a sub-object part demarcated by a fiat (not natural) boundary, which is about *dividing* a unified whole rather than *summing* discrete members. See `note-001`, `note-002`.

Exclusions: this crosswalk does not address Boundary/Site (a separate `0.2.0`-batch candidate; BFO's `fiat object part` and its demarcating "fiat surfaces" are directly relevant background for that future crosswalk, not resolved here) or Object (already `reviewed`; the Object/Aggregate contrast that BFO's, SUMO's, and UFO's own documentation each draw is noted but not re-litigated).

**Core.ttl placement is deliberately left open at this `draft` stage** — see Uncertainty.
