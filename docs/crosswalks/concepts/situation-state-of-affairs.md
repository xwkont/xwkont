## Scope Note

**Selection rationale (per `ADR-0018`'s admission bar):** Situation / State of Affairs is the fifth concept drafted from the `0.2.0` batch. `TODO.md`'s own seed flagged this candidate as "weaker/more UFO-centric than [Space, Abstract, Quantity]; good test of whether it generalizes or stays source-specific." An initial direct-verification pass (2026-07-04) found only **1 of 8 sources (GFO)** with an unambiguous formal class, and an independent external review reached the same HOLD conclusion, recommending DOLCE's DnS module and GFO's `Configuroid` be fetched before deciding this crosswalk's fate.

**Follow-up primary-source pass (2026-07-05)** fetched both: DOLCE's separate `ExtendedDnS.owl` module (`xwkont:ref:dolce-extendeddns-owl`) and the remainder of `modules/gfo-base.owl` plus GFO's dedicated `modules/situation/gfo-situation.ttl` module. Both resolved in favor of a materially stronger cross-source pattern than the initial pass found:

- **DOLCE** — contrary to the initial pass's "DOLCE has no formal situation class" finding (which only checked core `DOLCE-Lite.owl`), DOLCE's separate DnS module *does* formally define `#situation`, equivalent to `non-agentive-social-object ⊓ setting-for(some particular) ⊓ satisfies(some description)`. This is DOLCE's real treatment of the concept — core `DOLCE-Lite.owl`'s own commentary had said as much, but this pass is the first to independently fetch and verify it.
- **GFO** — the initial pass stopped at the parent class `Configuration` and missed a class literally named `Situation` (`rdfs:subClassOf Configuration`) in the same `gfo-base.owl` file, plus an entire dedicated `situation` module (`gfo-situation.ttl`) with its own `:Situation` class built on a different superclass lineage (`gfo-core:TimeExtendedIndividual`). GFO treats this concept seriously enough to warrant a whole extension module, not just a passing base-ontology class.
- **UFO** — unchanged from the initial pass: `ontouml:situation` is a genuine, well-documented OntoUML-vocabulary construct, though still not reified in the 2021 comprehensive paper's own prose.

Net effect: **3 of 8 sources (GFO, DOLCE, UFO)** now have an unambiguous formally-defined class matching this concept's core sense — DOLCE moves from "no correspondence, `state` is a different sense" to a genuine second solid correspondence. This is still short of `ADR-0018`'s original ≥3-source seed count in the strict sense of "named in the seed," but is now comparable in strength (not just source-count) to the batch's other concepts. See Uncertainty for the updated maintainer decision.

This crosswalk compares each source ontology's own treatment of a "situation"/"state of affairs"-like category — a configuration or circumstance comprehended as a whole, typically description- or context-dependent — distinct from Event/Process (already `reviewed`) and from Quality/Quantity's own notion of a value "holding" at a time. Sources compared: BFO, DOLCE, SUMO, UFO, GFO, YAMATO, TUpper, GUM (per `ADR-0015`'s 8-source scope).

**Cross-cutting finding, established this session:** the word "situation" is used informally, as a world- or time-index, in several sources' own prose (DOLCE's `state`/`process`/`stative` comments; UFO's 2021 paper) without ever being reified as a class there — this is a citation-precision trap distinct from, but related to, the ones found in `time.md`/`quantity-amount-of-matter.md`: informal usage of a term is not the same as that term being a defined category in the source, even when the source's own commentary explicitly gestures at the concept (DOLCE's comments describe what a "situation" would be — a description-dependent unifying criterion for a perdurant — while explicitly declining to formalize it in the fetched module; that formalization lives in DOLCE's separate DnS (Descriptions and Situations) extension, not core DOLCE-Lite). See `note-001`, `note-002`.

Exclusions: this crosswalk does not address Universal/Type (a separate `0.2.0`-batch candidate; UFO's OntoUML stereotype vocabulary — already the primary source for this crosswalk's UFO row — also defines `type`/`kind`/`phase` stereotypes relevant there) or Event/Process (already `reviewed`; DOLCE's `stative`/`state`/`process` branch is directly relevant background, already covered by `process.md` and `event.md` for the occurrent side of this comparison).

**Core.ttl placement is deliberately left open at this `draft` stage**, and is a live question of *whether* to place this concept at all, not just where — see Uncertainty.
