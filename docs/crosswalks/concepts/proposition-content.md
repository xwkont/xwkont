## Scope Note

**Selection rationale (per `ADR-0018`'s admission bar):** Proposition / Content is the ninth and final concept of the `0.2.0` batch. Direct primary-source verification this session found a nameable category in exactly the **3 of 8 sources** `TODO.md`'s seed claimed (DOLCE `proposition`, SUMO `Proposition`, YAMATO `proposition`/`content`) — the only `0.2.0`-batch concept so far where the seed's source count and source list both hold up unchanged. BFO, UFO, GFO, TUpper, and GUM were confirmed to have no reified Proposition/Content category; BFO's IAO extension (the standard BFO module for content-bearing/information entities, already relevant background for the `reviewed` `information-artifact.md`) was also checked and confirmed to have none.

This crosswalk compares each source ontology's own treatment of a truth-bearing or meaning-bearing abstract content — the *what is said/meant*, as distinct from any physical or informational bearer of it — closely related to but distinct from Information Artifact (already `reviewed`; SUMO's and YAMATO's own documentation each explicitly draw the bearer/content distinction this crosswalk's scope depends on) and from Abstract vs. Concrete (already drafted, `abstract-concrete.md`; both DOLCE's and SUMO's Proposition are direct subtypes of their own Abstract category). Sources compared: BFO, DOLCE, SUMO, UFO, GFO, YAMATO, TUpper, GUM (per `ADR-0015`'s 8-source scope).

**Cross-cutting finding, established this session:** all three confirmed sources converge on the same core distinction — a proposition/content is the abstract, representation-independent *thing meant*, sharply distinguished by each source's own documentation from any of its concrete/physical representations. SUMO's own documentation states this most explicitly, contrasting `Proposition` with `ContentBearingObject` by name: "A Proposition is a piece of information, e.g. that the cat is on the mat, but a ContentBearingObject is an Object that represents this information... an abstraction that may have multiple representations: strings, sounds, icons, etc." DOLCE's `proposition` similarly emphasizes "abstract content is purely combinatorial." YAMATO's own apparatus is the richest of the three, embedding the same bearer/content distinction directly into its representation theory: "Content is the hidden part of representation and it is a proposition which the author of the representation would like to convey through the representation... there can be many ways to code the same content into different forms." All three sources file this concept as (or, for YAMATO, alongside) their own Abstract category — the same convergence pattern `abstract-concrete.md` found for DOLCE/SUMO/GFO more generally, here holding for a third, more specific pair. See `note-001`.

Exclusions: this crosswalk does not address Information Artifact (already `reviewed`; the bearer/content distinction each source draws is the direct complement of that crosswalk's own scope, not re-litigated here) or Universal/Type (already drafted; SUMO's own `Class`/`Set` machinery, crosswalked there, is a structural sibling of `Proposition` under SUMO's `disjointDecomposition Abstract Quantity Attribute Relation Proposition List`, per `abstract-concrete.md`'s `note-004`).

**Core.ttl placement is deliberately left open at this `draft` stage** — see Uncertainty.
