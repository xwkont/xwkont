## Scope Note

**Selection rationale (per `ADR-0018`'s admission bar and a decision that each `0.2.0`-batch concept gets real rationale recorded as it is scoped, not backfilled):** Time is the first concept selected from the `0.2.0` batch. It clears `ADR-0018`'s ≥3-source threshold with the widest margin of any candidate in the batch — direct primary-source verification this session found a nameable temporal-region category in **6 of the 8 source ontologies** (all except UFO and YAMATO; UFO's absence is a confirmed finding in the fetched 2021 paper and the OntoUML vocabulary fallback, while YAMATO's is confirmed only as an absence in the one YAMATO document that is actually fetchable, not asserted as a settled fact about YAMATO overall — see Uncertainty). This corrects `docs/evaluations/foundational-ontology-concept-terms-matrix.md`'s "Observation 2," which had claimed only 5 sources (omitting SUMO and GUM, which this pass found do have Time content, and never checking GUM at all) — see that document's own updated Cross-Cutting Observation 2 and this crosswalk's Uncertainty section. Time is also methodologically load-bearing in a way none of the 8 already-`reviewed` concepts are: `continuant-occurrent.md`'s own core split is defined in terms of time ("persisting... through time" vs. "unfolding... through time") without Time itself ever being crosswalked as its own concept until now.

This crosswalk compares each source ontology's own treatment of time-as-a-category — the region, quantity, or class denoting temporal extent, instant, interval, or duration — not any single source's full philosophy of time (e.g., BFO's SNAP/SPAN framework, GFO's Brentano-based continuum theory, or UFO's process-philosophy commitments are each much larger literatures than what is scoped here). Sources compared: BFO, DOLCE, SUMO, UFO, GFO, YAMATO, TUpper, GUM (per `ADR-0015`'s 8-source scope).

**Cross-cutting finding, established this session:** there is no consensus across the 6 sources that have a Time category on *what kind of thing* Time is. BFO alone classifies its temporal region as a subtype of **Occurrent** — every other source that has a Time category treats it as **independent of the occurrent/continuant (or equivalent) split entirely**: DOLCE files it as an abstract `region` (a quality-space notion, the same branch as `physical-region`, neither endurant nor perdurant); SUMO files it under `Abstract`/`Quantity`, disjoint from `Physical` (where SUMO's `Process` sits); GFO gives it its own sui generis third top-level branch (`Space_time`, explicitly `owl:disjointWith` both `Presential` and `Occurrent`); TUpper makes `timepoint` one of four flat, co-equal top-level classes (alongside `object`/`activity`/`activity_occurrence`); GUM's `Time` has a dual parent (`Substance` and `TemporalObject`), an ancestor of neither its occurrent-like `Process` nor its continuant-like `SimpleThing`. This is the sharpest structural divergence found in this crosswalk — not a labeling difference but a genuine disagreement about where Time sits in each source's top-level architecture. See `note-001`.

Exclusions: this crosswalk does not address Spatial Region/Space (a separate `0.2.0`-batch candidate, planned separately, though several sources — BFO's `spatiotemporal region`, GFO's `Space_time` — treat Time and Space as siblings under a shared parent, noted but not resolved here) or the Abstract-vs-Concrete split (also a separate `0.2.0` candidate, though SUMO's and GFO's classification of Time on the abstract/non-concrete side is directly relevant background — see `note-001`).

**Core.ttl placement is deliberately left open at this `draft` stage.** Per the same process the original 8 concepts followed (a class is added to `data/ontology/core.ttl` only once its crosswalk reaches `reviewed`, consistent with `docs/governance/release-versioning-policy.md`'s `MINOR` = "reviewed crosswalk scope" definition), no `xwkont-core:Time` class exists yet. The cross-cutting finding above (5 of the 6 sources with a Time category — all except BFO — treat it as independent of Occurrent/Continuant) is a live open question for that eventual placement — see Uncertainty.
