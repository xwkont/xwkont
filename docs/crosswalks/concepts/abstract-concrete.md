# Abstract vs. Concrete

> **Local identifier:** `xwkont:concept:abstract-concrete`

> **Editorial status:** `reviewed`
> **Created:** `2026-07-04`
> **Modified:** `2026-07-06`

## Scope Note

**Selection rationale (per `ADR-0018`'s admission bar):** Abstract vs. Concrete is the third concept drafted from the `0.2.0` batch. Direct primary-source verification this session found an explicit, nameable Abstract (and, in some sources, Concrete) category in **5 of the 8 source ontologies** (DOLCE `abstract`, SUMO `Abstract`, GFO `Abstract`/`Concrete`, YAMATO `abstract`/`semi-abstract`, GUM `Abstraction`) — one more source than `TODO.md`'s seed claimed (the seed listed DOLCE, SUMO, GFO, YAMATO; this session's pass additionally found GUM's `Abstraction` class, previously uncredited). BFO and TUpper were confirmed to have no reified class of this kind; UFO was confirmed to have none at all (zero occurrences of "abstract" in the paper's full extracted text). See Source Definitions and Uncertainty.

This crosswalk compares each source ontology's own treatment of the abstract/concrete distinction as a *top-level classificatory split* — a class or disjoint-pair that sorts entities by whether they have a spatial/temporal location or physical embodiment — not any single source's full philosophy of abstracta (e.g., mathematical Platonism, nominalism about universals, or SUMO's own separate `SetOrClass` theory are each larger literatures than what is scoped here). Sources compared: BFO, DOLCE, SUMO, UFO, GFO, YAMATO, TUpper, GUM (per `ADR-0015`'s 8-source scope).

**Cross-cutting finding, established this session:** this is a higher-level split than any of the 8 original crosswalks or `time.md`/`space.md` — it is not itself a concept-in-the-BFO/DOLCE sense of a single class so much as a top-level *disjunction* that several sources make explicit and one (BFO) conspicuously does not. Four of the five sources with an Abstract category (DOLCE, SUMO, GFO, GUM) define it structurally the same way: the complement of having a spatial/temporal location or embodiment (DOLCE: "do not have spatial nor temporal qualities"; SUMO: "cannot exist at a particular place and time without some physical encoding or embodiment"; GFO: "independent from time and space"; GUM: "exists in metaphorical or qualitative space rather than in physical space"). YAMATO alone further subdivides the space: a `semi-abstract` category (needs only time, not space) sits between `physical` and fully `abstract`, a three-way split none of the other sources make explicitly. BFO's absence of any literal "Abstract" class is a real, load-bearing omission, not an oversight — BFO instead handles much of the same ground through `generically dependent continuant` (patterns/content that can have multiple concrete copies, e.g. a PDF file's content), which is still a Continuant with the capacity for concrete realization, not a category defined by the *absence* of spatiotemporal location the way the other five sources define Abstract. See `note-001`.

Exclusions: this crosswalk does not address Information Artifact (already `reviewed`; BFO's `generically dependent continuant` and DOLCE's `non-physical-endurant`/information-object branch are directly relevant background already covered there) or Proposition/Content (a separate `0.2.0`-batch candidate; several sources file `Proposition` as a subtype of their own Abstract category — noted below but not crosswalked here).

**Core.ttl placement is deliberately left open at this `draft` stage**, for the same reasons as `time.md`/`space.md`. No `xwkont-core:Abstract`/`xwkont-core:Concrete` split exists yet in `data/ontology/core.ttl`.

**Modeling-shape decision (recorded 2026-07-04, in response to external review):** this crosswalk records a recommendation — not a core.ttl edit, which still waits for `reviewed` status per this repository's governance model — that Abstract/Concrete be modeled as a **class pair**, a sibling-level split under `xwkont-core:Entity` alongside Continuant/Occurrent, not as a property or an editorial-only distinction. Rationale: (1) 5 of 8 sources (DOLCE, SUMO, GFO, YAMATO, GUM) define Abstract as a genuine formal class, not a property annotation, using the same criterion (absence of spatiotemporal location/embodiment); (2) `quantity-amount-of-matter.md` and `proposition-content.md` are now both drafted (this session), and both record SUMO/GFO subtypes of `Abstract` (`Quantity`, `Proposition`) as their own primary correspondences — a property-only or editorial-only treatment would leave those crosswalks' eventual core.ttl placements without a superclass to subclass from, whereas a class pair does not; (3) GFO's and SUMO's own formally exhaustive partitions (`owl:unionOf`/`disjointDecomposition`, `note-004`) are evidence this split functions the same way in source ontologies that Continuant/Occurrent already functions in XwkOnt's own core. This recommendation is not a settled fact about the source ontologies (unlike the crosswalk's other findings) — it is an XwkOnt-side modeling call, offered for maintainer confirmation at review time, per `uncertainty-001`.
