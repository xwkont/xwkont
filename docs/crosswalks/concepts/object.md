## Scope Note

This crosswalk compares BFO's `Object` (a subtype of `Material Entity`, itself under `Independent Continuant`), DOLCE's `Physical Object` (a subtype of `Physical Endurant`), UFO's `Substantial`/`Kind` (an independent endurant further typed by sortals), and SUMO's `Object` (a subtype of `Physical`). It builds directly on `docs/crosswalks/concepts/continuant-occurrent.md`: Object sits on the continuant/endurant side of each source ontology's top-level split.

**Correction (2026-07-01), following direct primary-source verification:** this crosswalk originally claimed "UFO does not have a class literally named 'Object.'" That claim was **wrong** and has been corrected after fetching and manually extracting text from UFO's actual 2021 Applied Ontology paper. UFO's primary source states: **"Objects (aka functional complexes) are entities whose parts play differentiated functional roles with respect to the whole, e.g., a human body, an organization, a computer, John's car."** `Object` is a genuine, formal leaf category of `Substantial`, alongside `Quantity` (homeomerous amounts of matter) and `Collective` (parts playing the same role, e.g. a forest of trees). The earlier "Substantial/Kind vs. Object" framing conflated two different UFO distinctions: `Substantial`/`Moment` is the independent/dependent-endurant split (where Object lives on the Substantial side, alongside Quantity and Collective), while `Kind` is a *sortal classification* that can apply to any of these leaf categories (an `ObjectKind`, `QuantityKind`, etc. are explicitly paired types in UFO's own formal apparatus). See Source Definitions and Uncertainty for the corrected picture.

Exclusions: this crosswalk does not cover Quality, Role, or Relation (separate concepts in `data/ontology/core.ttl`), and does not attempt to resolve individuation-criteria debates (e.g., what exactly grounds an object's persistence conditions) in full philosophical depth — see Uncertainty.

**Update (2026-07-02), GFO/YAMATO/TUpper/GUM buildout:** per the pattern `continuant-occurrent.md` established, this crosswalk now also covers the four `ADR-0015` sources, reusing the OWL/CLIF/technical-report artifacts already fetched for that crosswalk. GFO's Object-equivalent is `Material_object` (a subclass of `Discrete_presential`, itself under `Presential`) — GFO's `Material_persistant` class explicitly glosses this lineage as "related to those entities which are called sometimes continuants or objects, as apples, cars or houses," a rare case of a source ontology using XwkOnt's own vocabulary in its own definition text. YAMATO's Object is defined relationally, as "the interface between its internal and external processes" (quoting Galton 2009's "the water falls, but the waterfall doesn't fall" framing) rather than by an intrinsic unity criterion like BFO/DOLCE. TUpper's `object` (already fetched for `continuant-occurrent.md`) is reused unchanged — same negative-definition term, same low-confidence structural-analog treatment as SUMO. GUM's Object-equivalent is `DecomposableObject`/`NonDecomposableObject`, both direct subclasses of `SimpleThing` — a linguistic (parts-mentioned-in-discourse) criterion, not a metaphysical one.

**Epistemic note:** BFO and SUMO content is verified directly against primary sources (`bfo-core.ttl`, `Merge.kif`). UFO content is now also directly verified (2026-07-01) via manual PDF-stream extraction of the actual 2021 Applied Ontology paper — no PDF-text-extraction tool is otherwise available in this environment, so a custom `zlib`-based extractor was used, the same technique that worked for a DOLCE 2022 paper elsewhere in this repository. DOLCE's own Physical Object content in *this* crosswalk remains secondary-sourced (the manual-extraction pass focused on Role/Quality-relevant DOLCE passages and all of UFO; DOLCE's Physical Object specifically was not re-checked against the extracted D18 text in this pass — see Future Work).
