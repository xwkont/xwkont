## Scope Note

**Selection rationale (per `ADR-0018`'s admission bar):** Spatial Region/Space is the second concept drafted from the `0.2.0` batch, following `time.md`. It clears `ADR-0018`'s ≥3-source threshold: direct primary-source verification this session found a nameable spatial-region category in **6 of the 8 source ontologies** (BFO, DOLCE, SUMO, GFO, TUpper, GUM — the same six that have a Time category, minus UFO and YAMATO, which this session confirmed lack both). `TODO.md`'s seed description ("BFO `spatial region` + 4 dimensional subtypes, DOLCE `physical-region`/`space-region`, GFO `Space`, TUpper `MaterialObject`/`ShapedObject`/`VoluminalRegion`/`ArealRegion`/`Curve`, SUMO `Region` filed under `Object`") is confirmed in its structural core but corrected on two points of detail — see Source Definitions and `uncertainty-002`.

This crosswalk compares each source ontology's own treatment of space-as-a-category — the region, extent, or class denoting spatial extension, location, or dimensionality — not any single source's full philosophy of space (e.g., BFO's four-dimensionalist spacetime framework or GFO's Brentano-based space model are larger literatures than what is scoped here). Sources compared: BFO, DOLCE, SUMO, UFO, GFO, YAMATO, TUpper, GUM (per `ADR-0015`'s 8-source scope).

**Cross-cutting finding, established this session:** unlike Time — where 5 of 6 sources with a Time category converged on classifying it as independent of the occurrent/continuant split, with BFO the lone holdout (`time.md`'s `note-001`) — Space shows **no comparable majority pattern**. BFO classifies its `spatial region` as a **Continuant** (specifically `immaterial entity` → `independent continuant` → `continuant`), the structural mirror-opposite of BFO's own `temporal region` (Occurrent). DOLCE files `space-region` as an independent abstract region, a sibling of `temporal-region` under the same `region`/`abstract` branch — matching Time's classification pattern exactly. SUMO files `Region` under `Object` → `Physical`, the opposite side of its `Physical`/`Abstract` split from `TimeMeasure` (Abstract) — mirroring BFO's opposition, but via a different top-level distinction. GFO makes `Space` an explicit sibling of `Time` under the shared `Space_time` branch — the same pattern GFO showed for Time. TUpper's spatial predicates are not promoted to top-level flat-class status the way `timepoint` is — a structural asymmetry Time did not have. GUM gives `Space` the same dual-parented, `SpatialTemporal`-descended structure as `Time` — see `note-001` and `uncertainty-001` for a correction this session found to `time.md`'s own prior GUM finding. See `note-002`.

Exclusions: this crosswalk does not address the Abstract-vs-Concrete split (a separate `0.2.0`-batch candidate, though SUMO's and GFO's placement of Space on the concrete/physical side is directly relevant background) or Boundary/Site (also a separate `0.2.0` candidate; BFO's `site` and GFO's `Spatial_boundary`/`Surface` are closely related but distinct categories, noted below but not crosswalked here).

**Core.ttl placement is deliberately left open at this `draft` stage**, for the same reasons as `time.md` (`uncertainty-001` there) — no `xwkont-core:Space` class exists yet in `data/ontology/core.ttl`. The classification divergence documented above (BFO: continuant; DOLCE/GFO/GUM: independent/sibling category; SUMO: physical/concrete) is a live open question for that eventual placement, made harder by the fact that Time and Space themselves don't even converge on the *same* divergence pattern — see Uncertainty.
